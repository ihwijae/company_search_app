# 공용폴더 운영 정리

## 목적

로컬 PC마다 따로 파일을 들고 쓰는 방식 대신, 구글드라이브 데스크탑의 공용 폴더를 기준으로 다음 데이터를 팀원끼리 공유하도록 정리했다.

- 업체조회 엑셀 파일
- 협정보드 저장 JSON
- 실적관리 DB 및 첨부
- 메일 주소록 JSON

## 공용 루트 폴더

기준 폴더는 각 PC의 구글드라이브 데스크탑에 잡히는:

- `...\내 드라이브\협정보조툴`
- 또는 영문 환경이면 `...\My Drive\협정보조툴`

중 하나다.

중요한 점:

- 드라이브 문자는 고정이 아니다.
- 어떤 PC는 `F:\내 드라이브\협정보조툴`
- 어떤 PC는 `J:\내 드라이브\협정보조툴`

일 수 있다.

코드는 드라이브 문자를 고정하지 않고, `내 드라이브` 또는 `My Drive`를 먼저 찾은 뒤 그 아래 `협정보조툴` 폴더를 기준으로 경로를 계산하도록 수정했다.

## 현재 공유 구조

### 1. 업체조회 엑셀

업체조회는 `협정보조툴` 폴더 아래에서 전기/통신/소방 엑셀을 자동 탐색한다.

규칙:

- 파일명에 `전기`, `통신`, `소방` 키워드가 반드시 포함되어야 한다.
- 같은 타입 파일이 여러 개 있으면 최신 수정 파일을 우선 사용한다.
- 앱 실행 중에도 폴더를 감시해서 파일 내용 변경, 새 파일 추가, 기존 파일 삭제를 자동 반영한다.

수동 경로 설정도 남겨뒀다.

- 기본은 자동(`auto`)
- 사용자가 업로드/경로설정으로 직접 지정하면 수동(`manual`)
- 수동 지정 후에도 같은 폴더 안에서 파일명이 날짜 기준으로 바뀌면 최신 파일을 따라간다.

업로드 드로어에는 현재 사용 파일명과 `자동 연결` / `수동 지정` 상태가 보이도록 수정했다.

### 2. 협정보드

협정보드 기본 저장 폴더는:

- `...\협정보조툴\협정`

이다.

협정 저장과 불러오기는 같은 루트를 사용한다.

주의:

- 예전에 다른 협정보드 경로를 쓰던 사용자 설정은 강제로 새 기본 경로로 맞추도록 수정했다.
- 앱 재실행 시 공용 협정 폴더 기준으로 다시 맞춰진다.

### 3. 실적관리

실적관리 저장 루트는:

- `...\협정보조툴\실적관리`

이다.

여기에 저장되는 것:

- `records.sqlite`
- 첨부 폴더
- 관련 메타 파일

실적을 등록하거나 첨부를 추가하면 이 공용 경로에 바로 저장된다.

fallback:

- 구글드라이브가 없는 환경이면 실적관리는 로컬 `userData` 경로로 fallback 한다.
- 따라서 공유가 목적이면 그 PC는 반드시 구글드라이브가 연결돼 있어야 한다.

실적관리 UI 쪽 추가 수정:

- 법인 필터에 `전체` 추가
- 공사종류 패널 스크롤 추가
- 페이지 번호는 20개 단위로만 보이게 수정
- 공사명 앞의 `공사명:` 접두어 제거
- 긴 실적 내용이 있어도 본문 영역만 스크롤되도록 레이아웃 수정

### 4. 메일 주소록

메일 주소록 기준 파일은:

- `...\협정보조툴\mail-addressbook.json`

이다.

동작:

- 메일 화면 진입 후 주소록은 이 JSON 파일을 기준으로 로드
- 앱에서 주소록 수정 시 자동 저장
- 외부에서 JSON 파일을 수정해도 앱이 다시 읽어오도록 감시 추가

fallback:

- 구글드라이브가 없으면 로컬 `userData/mail-addressbook.json` 사용

예전 로컬 `mail:addressBook`은 브라우저/앱 로컬 저장소 키였고, 현재 기준 원본은 JSON 파일이다.

## 추천 폴더 구조

아래처럼 정리해서 쓰는 것을 권장한다.

```text
내 드라이브
└─ 협정보조툴
   ├─ 협정
   ├─ 실적관리
   ├─ mail-addressbook.json
   └─ search-data
      ├─ 1.협력업체요약(2024년전기)2026.03.31.xlsx
      ├─ 1.협력업체요약(2024년통신)2026.03.31.xlsx
      └─ 1.협력업체요약(2024년소방)2026.03.31.xlsx
```

참고:

- 엑셀 파일은 `협정보조툴` 아래 어디에 있어도 현재 탐색되지만
- `search-data` 같은 전용 폴더로 모아두는 것이 탐색 정확도와 관리 측면에서 더 좋다.

## 운영상 주의점

### 1. 동시 수정 충돌

공용 파일 구조이므로 동시 수정 충돌 가능성이 있다.

- 협정보드 JSON
- 실적관리 DB/첨부
- 메일 주소록 JSON

특히 같은 파일을 동시에 수정하면 마지막 저장본 기준으로 덮일 수 있다.

### 2. 구글드라이브 동기화 지연

자동 반영은 즉시가 아니라 “구글드라이브 동기화 완료 후” 반영된다고 보는 것이 맞다.

### 3. fallback 환경

실적관리, 메일 주소록은 구글드라이브가 없는 환경에서 로컬 fallback 이 있다.

하지만 이 경우:

- 수정 내용은 공용 폴더에 공유되지 않는다.

즉 팀 공유가 목적이면 구글드라이브 데스크탑이 반드시 연결돼 있어야 한다.

## 다음 세션에서 확인할 것

- 다른 드라이브 문자(`F:`가 아닌 `J:` 등) 환경에서 실제로 정상 탐지되는지
- `협정보조툴` 하위 폴더 구조를 팀 운영 기준으로 확정할지
- 업체조회 엑셀 자동 탐색 범위를 `협정보조툴` 전체가 아닌 `search-data` 같은 하위 폴더로 더 좁힐지
- 주소록/협정보드/실적관리의 동시 수정 충돌 대응을 추가할지
